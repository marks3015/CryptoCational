# Contributing / Contribuição

## English
Thank you for your interest in contributing to **CryptoCational**! We welcome contributions that improve the cryptographic core, refine the UI, or expand our documentation.

### 1. How to Contribute
1. **Fork the Repository**: Create a personal copy of the project on your GitHub account.
2. **Clone Locally**: Run `git clone https://github.com/your-username/interface.git`.
3. **Environment Setup**: Install dependencies using `pip install -r requirements.txt`.

### 2. Branching Strategy
We use a simple branching model to keep the project organized:
- **Main Branch**: Only contains stable, production-ready code.
- **Feature Branches**: All new work should be done on a separate branch.
  - Naming convention: `feature/short-description` or `fix/issue-description`.
  - Example: `git checkout -b feature/caesar-cipher`

### 3. Submission Process (Pull Request)
1. **Commit Your Changes**: Use clear, English commit messages (e.g., `feat: add caesar cipher implementation`).
2. **Push to GitHub**: `git push origin feature/caesar-cipher`.
3. **Open a Pull Request (PR)**:
   - Navigate to the original repository on GitHub.
   - Click "New Pull Request".
   - Describe your changes clearly and link any related issues.
4. **Review**: Maintainers will review your code. Be prepared to make adjustments based on feedback.

## Português
Obrigado pelo seu interesse em contribuir para o **CryptoCational**! Aceitamos contribuições que melhorem o core criptográfico, refinem a UI ou expandam nossa documentação.

### 1. Como Contribuir
1. **Fork do Repositório**: Crie uma cópia pessoal do projeto em sua conta no GitHub.
2. **Clone Localmente**: Execute `git clone https://github.com/seu-usuario/interface.git`.
3. **Configuração do Ambiente**: Instale as dependências usando `pip install -r requirements.txt`.

### 2. Estratégia de Branching
Usamos um modelo simples de branching para manter o projeto organizado:
- **Branch Principal (Main)**: Contém apenas código estável e pronto para produção.
- **Branches de Funcionalidades (Feature Branches)**: Todo novo trabalho deve ser feito em uma branch separada.
  - Convenção de nome: `feature/descricao-curta` ou `fix/descricao-do-problema`.
  - Exemplo: `git checkout -b feature/cifra-de-cesar`

### 3. Processo de Envio (Pull Request)
1. **Commit de Suas Alterações**: Use mensagens de commit claras em inglês (ex: `feat: add caesar cipher implementation`).
2. **Push para o GitHub**: `git push origin feature/cifra-de-cesar`.
3. **Abra um Pull Request (PR)**:
   - Navegue até o repositório original no GitHub.
   - Clique em "New Pull Request".
   - Descreva suas alterações claramente e vincule quaisquer problemas (issues) relacionados.
4. **Revisão**: Os mantenedores revisarão seu código. Esteja preparado para fazer ajustes com base no feedback.
